# 커밋 메시지 규칙

## 형식

```
<type>(<scope>): <한국어 제목>

- <변경사항 1>
- <변경사항 2>
```

## Type

- `feat`: 새 기능
- `fix`: 버그 수정
- `refactor`: 리팩토링
- `chore`: 유지보수 (빌드, 설정, 의존성)
- `docs`: 문서
- `test`: 테스트
- `style`: 코드 스타일

## Scope 예시

- 기능/컴포넌트: `button`, `post`, `about`, `contact`
- 의존성: `deps`
- 도구/설정: `husky`, `kiro`
- 모듈/레이어: `shared`, `markdown`

## 원칙

1. 제목과 본문 모두 한국어
2. 간결하게, 핵심만
3. 변경사항은 bullet point로 나열
